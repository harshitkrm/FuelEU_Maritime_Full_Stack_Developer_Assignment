# Reflection: Building FuelEU with Cursor

This project mixed maritime regulation, financial-style penalties, and a non-trivial pooling algorithm inside a strict hexagonal layout. Using Cursor for that work changed where time went: less on boilerplate and file wiring, more on getting formulas and boundaries right.

**Efficiency where it mattered.** Article 21 pooling is a good example. A greedy allocation with three business rules (aggregate feasibility, “no worse off” for deficit ships, protection for non-deficit ships) is easy to describe in prose and painful to get correct in one pass. In a traditional workflow you might sketch the algorithm, implement, debug with ad-hoc logs, and only then add tests. Here, the assistant could iterate quickly on the allocator, add focused unit tests, and fix a real bug (initial version did not conserve total compliance balance across members). The cost of trying a wrong approach dropped sharply because refactors stayed localized to pure functions in `src/core/domain`.

**Cursor vs “manual” for pooling logic.** Manually, you still have to *understand* the regulation and choose a representation (per-ship CB, sorted order, remainder split). The gain is not that the model invents the algorithm from nothing—it is that implementation and test feedback loops are shorter, so mistakes surface in minutes instead of sessions. The risk is over-trust: a plausible-looking formula or a clever one-off fix can look authoritative. That is why we anchored the consecutive penalty and pooling behaviour with explicit tests and, in this repo, an honest workflow log when something had to be corrected (e.g. the consecutive multiplier applied to the base penalty as specified, not an informal compound model).

**Hexagonal architecture and AI.** Ports and adapters are a natural fit for collaboration with an agent: “domain here, Prisma there, never the twain shall meet” is a rule the tools can enforce consistently. The main discipline is human: review inbound adapters to ensure they only call use-cases, and review outbound adapters to ensure they only satisfy ports. Cursor helped generate repetitive repository and HTTP glue; the value is preserved only if those layers stay thin.

**Bottom line.** For this stack, Cursor was most valuable on **regulatory math + invariants + tests**, and on **scaffolding** the frontend tabs and API client. The remaining work—judging whether the pool rules match the assignment, naming, and operational concerns like `DATABASE_URL` and CORS—is still engineering judgment. Used that way, the tool is a multiplier on implementation speed, not a substitute for domain verification.
